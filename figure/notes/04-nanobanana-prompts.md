## 核心 Prompt 模板

### 基础模板结构

```
[图表类型] + [风格描述] + [具体内容] + [配色要求] + [输出规格]
```

### 万能科研图 Prompt

```
请生成一张专业的科研机制图：
- 风格：期刊机制图/综述封面风格
- 内容：[你的具体内容]
- 配色：统一3-4个主色，避免过度渐变
- 要求：清晰注释，箭头方向明确
- 格式：竖版4K
```

## 分场景 Prompt 整理

### 场景一：方法流程图 (Method Pipeline)

```
生成一张深度学习方法流程图：
- 从左到右的Pipeline布局
- 输入 → 预处理 → 模型 → 输出
- 每个模块用不同颜色区分
- 箭头清晰，标注模块名称
- 风格简洁，适合顶会论文
```

**变体 - 含数据流**：
```
生成方法流程图，要求：
- 显示数据维度变化 (如 224×224 → 7×7 → 1000)
- 每层操作用图标表示
- Conv、Pool、FC 用不同形状
- 标注参数量或计算量
```

### 场景二：模型架构图 (Architecture)

```
绘制Transformer架构图：
- 标准编码器-解码器结构
- Multi-Head Attention 模块高亮
- 残差连接用虚线表示
- 配色参考 Nature/Science 风格
- 每个子模块有清晰边界
```

**变体 - Vision Transformer**：
```
绘制ViT架构图：
- 图像分块 → Patch Embedding → Transformer Encoder → 分类头
- 显示位置编码的添加方式
- CLS token 单独标注
- 整体风格学术简洁
```

### 场景三：综述框架图 (Survey Overview)

```
生成综述论文的方法分类框架图：
- 树状/层级结构
- 顶层是研究领域
- 中层是主要方向（3-4个分支）
- 底层是具体方法
- 每个分支用不同颜色
- 适合放在综述Introduction
```

**变体 - 时间线综述图**：
```
绘制领域发展时间线：
- 横轴为时间（2017-2024）
- 纵轴为不同方向
- 里程碑论文用图标标注
- 显示方法演进关系
- 风格清晰美观
```

### 场景四：实验对比图 (Comparison)

```
生成方法对比示意图：
- 左右/上下对比布局
- 方法A vs 方法B vs Ours
- 用图标展示各自优缺点
- 我们的方法用绿色/高亮标注优势
- 适合 Method 或 Introduction
```

### 场景五：多模态流程图

```
绘制多模态学习流程图：
- 图像分支 + 文本分支 + 融合模块
- 每个模态用不同颜色
- 特征融合处用箭头汇聚
- 显示特征维度对齐
- 风格统一，标注清晰
```

### 场景六：强化学习框架图

```
生成强化学习交互框架图：
- Agent - Environment 循环
- State, Action, Reward 流向清晰
- Policy/Value Network 内部结构
- 箭头标注数据含义
- 配色温和，易于理解
```

## 高级技巧 Prompt

### 技巧一：控制配色

```
配色要求：
- 主色：#4A90D9（科技蓝）
- 辅色：#50C878（清新绿）
- 强调：#FF6B6B（警示红）
- 背景：纯白 #FFFFFF
- 所有文字使用深灰 #333333
```

### 技巧二：控制风格

**期刊风格**：
```
风格：Nature/Science 期刊插图风格
- 线条精细，配色典雅
- 无卡通元素
- 专业学术感
```

**手绘风格**：
```
风格：手绘漫画科研图
- 线条略带手绘感
- 配色活泼但不花哨
- 适合轻松展示
```

**扁平化风格**：
```
风格：扁平化设计
- 无阴影无渐变
- 纯色块填充
- 现代简约感
```

### 技巧三：控制布局

```
布局要求：
- 从左到右流程布局
- 主体居中，留白适中
- 预留标注空位
- 比例 16:9（PPT适用）/ 4:3（论文适用）
```

### 技巧四：迭代修改 Prompt

```
在上一张图的基础上：
- 只修改配色，不改变主体结构
- 将蓝色系改为绿色系
- 加粗箭头
- 放大模块内文字
```

## 使用流程

```
Step 1: 准备内容
├── 列出所有模块/概念
├── 确定它们之间的关系
└── 画草图理清布局

Step 2: 选择模板
├── 根据场景选择 Prompt
├── 填入具体内容

Step 3: 生成初版
├── 提交给 Nano Banana Pro
├── 查看效果
└── 记录需要调整的地方

Step 4: 迭代优化
├── 用"只修改xxx"的Prompt微调

Step 5: 导出使用

├── 如有小瑕疵用PPT/PS修复
└── 导出PDF用于LaTeX
```

## 常见问题

### Q: Prompt 写很长但效果不好？
A:
- 分步骤描述，不要堆砌
- 先生成大框架，再微调细节
- 关键词优先，修饰词其次

### Q: 每次生成结果差异很大？
A:
- 描述更具体（如"3个模块"而非"几个模块"）
- 指定参考风格或颜色
- 使用"在上一张基础上修改"保持一致性

### Q: 生成的图有乱码/文字错误？
A:
- Prompt 中写"不要写字，只画结构"
- 后期用 PPT 添加正确标注
- 或者只生成图标/符号
