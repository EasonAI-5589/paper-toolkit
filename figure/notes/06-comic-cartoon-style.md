## 漫画风绘图核心技巧

### 1. 线条风格

```
特点：
- 线条略带手绘感，不完全规整
- 圆角矩形代替直角
- 箭头用弯曲线条
- 整体轻松活泼
```

### 2. 配色原则

```
- 主色 1-2 种（蓝/绿/紫）
- 辅助色用于区分模块
- 背景纯白或浅灰
- 避免高饱和度撞色
```

**推荐配色**：
| 风格 | 主色 | 辅色 | Hex |
|------|------|------|-----|
| 清新蓝 | 天蓝 | 浅橙 | #74b9ff, #ffeaa7 |
| 温暖橙 | 珊瑚 | 浅蓝 | #fab1a0, #81ecec |
| 自然绿 | 薄荷 | 淡紫 | #00b894, #a29bfe |

### 3. Icon 使用

```
- 用图标代替文字说明
- 保持图标风格统一
- 推荐：Flaticon, IconFont, Noun Project
- 尺寸一致，间距均匀
```

### 4. 布局原则

```
- 留白充足，不要填满
- 从左到右或从上到下
- 模块之间间距一致
- 重要内容居中或放大
```

## Nano Banana 漫画风 Prompt

### 基础漫画风

```
生成一张漫画风格的LLM框架图：
- 线条略带手绘感，不规整
- 模块用圆角矩形，有轻微阴影
- 配色温暖柔和（蓝+橙为主）
- 用图标表示核心概念
- 背景纯白，整体简洁
- 不要写字，只画结构和图标
```

```

- 适合技术论文的Method部分
```

## 注意事项

1. **保持学术性** - 漫画风不等于幼稚，内容要专业
2. **风格统一** - 一篇论文所有图保持同一风格
3. **可读性优先** - 不要为了好看牺牲清晰度
4. **适度装饰** - 图标和装饰不要喧宾夺主
